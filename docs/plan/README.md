# docs/plan 目录规范

本目录存放项目迭代计划文档。所有计划文件须遵循以下命名与内容规范。

---

## 文件命名规范

| 类型 | 格式 | 示例 |
|---|---|---|
| 主计划 | `NNN-描述性名称.md` | `001-align-user-py-to-so.md` |
| 子计划/补充清单 | `NNN-子主题.md` | `009-tls-exceptions.md` |

编号 `NNN` 为三位数字，按创建顺序递增，全局唯一，不回收。

---

## 文件内容规范

### 1. 标题

- 主计划：`# Plan NNN: 中文或英文标题`
- 子计划：`# NNN 子标题`

### 2. 目标引用块

标题下方紧跟一个 `> 目标：...` 引用块，一句话概括本计划的核心目标。禁止展开逻辑和细节。

### 3. 分隔线

目标引用块后使用 `---` 分隔线，再进入正文。

### 4. 章节编号

| 层级 | 格式 |
|---|---|
| 一级章节 | `## 一、...` 或 `## 1. ...` |
| 二级章节 | `### 1.1 ...` |
| 三级章节 | `#### 1.1.1 ...` |

同一文件内须保持编号风格一致（中文数字或阿拉伯数字二选一）。

### 5. 表格

状态对比、配置项、影响分析、验收清单等须使用 Markdown 表格，表头左对齐，内容对齐保持一致。

### 6. 代码块

- 代码示例须标注语言：`python`、`yaml`、`dockerfile`、`bash`、`diff` 等
- 展示修改优先使用 `diff` 代码块
- 命令示例须是可复制的单行或多行命令

### 7. 关键信息标记

- 强调：`**加粗**`
- 重要提示：`> **注意**：...` 或 `> **关键约束**：...` 引用块
- 优先级：`[P0]`、`[P1]`、`[P2]`
- 风险等级：**高** / **中** / **低**

### 8. 任务描述结构

涉及具体改动的任务须包含：

- `**文件**: path/to/file`
- `**操作**: ...`
- 如需验收：`**验收命令**：` + `bash` 代码块

### 9. 依赖关系

多任务/多 Phase 计划须显式说明串行或并行关系，可使用：

- 树形图（ASCII 字符画）
- 依赖符号：`//`（并行）、`‖`（串行）
- 表格：`| 文件 | 涉及任务 | 处理方式 |`

### 10. 附录

参考性长内容（如常量表、枚举值、调用链）放至末尾附录，不混入主线章节。
